# エージェント道場

BlueStarにおけるAIエージェントの定義、設計、および設定を一元管理するリポジトリ「道場」です。
特定のAIツール（Gemini CLI, Claude Code, Cursor, GitHub Copilot等）に依存しない「コア定義」を管理し、そこから実用的な設定へと昇華させます。

[**エージェント一覧を確認する (AGENTS.md)**](./AGENTS.md)

## 道場の理念：プログレッシブディスクロージャー
ユーザーはいきなり詳細なエージェントを使い分ける必要はありません。まずは「師範」に指示を出し、必要に応じて専門的な「弟子」や「道具」が呼び出される、段階的な情報開示の構造を目指します。

## ディレクトリ構成と役割

本リポジトリは、**「正規版（Stable）」**と**「実験版（Experimental）」**の2つのラインで構成されています。

| 役割 | 正規・安定版 (Stable) | 実験・開発版 (Experimental) |
| :--- | :--- | :--- |
| **AIエージェント定義** | **`shihan/`** (師範)<br>正式採用されたプロンプトと設定。<br>ここにあるものをコピーして利用します。 | **`deshi/`** (弟子)<br>開発中・修行中のプロンプト。<br>新しいアイデアや機能の実験場です。 |
| **ドキュメント・知識** | **`toranomaki/`** (虎の巻)<br>全体設計、ルール、名鑑など、<br>合意形成済みの正本ドキュメント。 | **`zatsunomaki/`** (雑の巻)<br>壁打ちメモ、思考ログ、草稿。<br>清書される前のアイデア置き場です。 |

### その他のディレクトリ
- **`menkyokaiden/`** (免許皆伝): 自然言語の定義を、Gemini CLIやCursorなどの実際のツール設定ファイルに変換した成果物です。

## 育成フロー (Lifecycle)
1. **雑記 (`zatsunomaki/`)**: アイデアを出し、AIと壁打ちを行う。
2. **修行 (`deshi/`)**: プロンプトを作成し、実動作をテストする。
3. **昇格 (`shihan/` & `toranomaki/`)**: 十分な品質になれば、正規ディレクトリへ移動（マージ）する。
4. **適用 (`menkyokaiden/`)**: 最終的にツール用設定ファイルとして出力・配布する。
