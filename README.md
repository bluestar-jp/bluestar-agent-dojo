# エージェント道場

BlueStarにおけるAIエージェントの定義、設計、および設定を一元管理するリポジトリ「道場」です。
特定のAIツール（Gemini CLI, Claude Code, Cursor, GitHub Copilot等）に依存しない「コア定義」を管理し、そこから実用的な設定へと昇華させます。

[**エージェント一覧を確認する (AGENTS.md)**](./AGENTS.md)

## 道場の理念：プログレッシブディスクロージャー
ユーザーはいきなり詳細なエージェントを使い分ける必要はありません。まずは「師範」に指示を出し、必要に応じて専門的な「弟子」や「道具」が呼び出される、段階的な情報開示の構造を目指します。

## ディレクトリ構成

### `makimono/` (巻物)
道場全体の設計図、運用ルール、エージェント名鑑を管理します。
- `grand-design.md`: 全体設計・エージェント相関図
- `curriculum.md`: 修行から免許皆伝への昇格基準
- `catalog.md`: エージェント一覧

### `shihan/` (師範)
正式に採用された、信頼できるエージェントの定義（Single Source of Truth）です。
- `prompts/`: 各師範のシステムプロンプト
- `okite/`: 【掟】全エージェントが遵守すべき共通憲法 (Constitution)
- `hiden/`: 【秘伝】組織の知識・コンテキスト (Context)

### `deshi/` (弟子)
新しいエージェントの育成や、実験的なアイデアを試すエリアです。
- `candidates/`: 師範代への昇格を待つプロンプト候補
- `sandbox/`: 開発者が自由に稽古を行う実験場

### `menkyo-kaiden/` (免許皆伝)
師範の教えを、各AIツールの仕様に合わせて変換・適用した設定ファイル群です。
- `gemini-cli/`: .gemini/ 用の設定
- `claude-code/`: Claude Code用の設定
- `cursor/`: .cursorrules 等

## 育成フロー
1. **修行 (`deshi/sandbox`)**: 自由にプロンプトを書き、挙動を試す。
2. **審査 (`deshi/candidates`)**: 品質や掟への準拠を確認し、PRを作成。
3. **師範代 (`shihan/`)**: マージされ、正式なエージェントとして登録。
4. **免許皆伝 (`menkyo-kaiden/`)**: 各ツールで利用可能な設定として配布。
