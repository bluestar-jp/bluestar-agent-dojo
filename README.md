# エージェント道場

BlueStarにおけるAIエージェントの定義、設計、および設定を一元管理するリポジトリ「道場」です。
特定のAIツール（Gemini CLI, Claude Code, Cursor, GitHub Copilot等）に依存しない「コア定義」を管理し、そこから実用的な設定へと昇華させます。

[**エージェント一覧を確認する (AGENTS.md)**](./AGENTS.md)

## 道場の理念：プログレッシブディスクロージャー
ユーザーはいきなり詳細なエージェントを使い分ける必要はありません。まずは「師範」に指示を出し、必要に応じて専門的な「弟子」や「道具」が呼び出される、段階的な情報開示の構造を目指します。

## ディレクトリ構成

### `toranomaki/` (虎の巻)
道場全体の設計図、運用ルール、エージェント名鑑を管理します。
- `grand-design.md`: 全体設計・エージェント相関図
- `curriculum.md`: 修行から免許皆伝への昇格基準
- `catalog.md`: エージェント一覧

### `zatsu-no-maki/` (雑の巻)
壁打ちメモ、ブレスト、アイデアの断片などの草稿エリアです。
将来的に虎の巻へ清書される前の、思考のログ置き場として利用します。

### `shihan/` (師範)
正式に採用された、信頼できるエージェント設定のセット（Stable Channel）です。
このディレクトリ配下をツール設定にコピーするだけで利用可能です。
- `agents/`: 各師範のシステムプロンプト定義
- `rules/`: 【掟】全エージェントが遵守すべき共通憲法 (Constitution)
- `knowledge/`: 【秘伝】組織の知識・コンテキスト (Context)
- `skills/`: エージェントが利用可能なツールやスキルの定義

### `deshi/` (弟子)
新しいエージェントの育成や、実験的なアイデアを試すセット（Experimental/Beta Channel）です。
構成は `shihan/` と完全に同一であり、開発中の機能をここでテストします。
- `agents/`: 開発中・修行中のプロンプト
- `rules/`: 試験的なルール
- `knowledge/`: 作成中のナレッジ
- `skills/`: テスト中のスキル

### `menkyo-kaiden/` (免許皆伝)
師範の教えを、各AIツールの仕様に合わせて変換・適用した設定ファイル群です。
- `gemini-cli/`: .gemini/ 用の設定
- `claude-code/`: Claude Code用の設定
- `cursor/`: .cursorrules 等

## 育成フロー
1. **修行 (`deshi/`)**: `deshi/agents/` 等で新しいプロンプトやルールを作成し、実際の挙動を試す。
2. **審査 (Review)**: 品質や掟への準拠を確認し、PRを作成。
3. **昇格 (`shihan/`)**: マージされ、正式なエージェントとして `shihan/` へ移動。
4. **免許皆伝 (`menkyo-kaiden/`)**: 各ツールで利用可能な設定として配布。
