# Use Case Testing Guide

- Purpose: ユースケーステストの詳細ガイド（ISTQB準拠）
- Scope: フローの記述方法、アクター分析、網羅基準

## 概要

ユースケーステストは、アクターとシステムの対話をシナリオとして記述し、基本フロー・代替フロー・例外フローからテストケースを導出する技法である。

## ユースケースの構成要素

| 要素 | 説明 |
| --- | --- |
| ユースケース名 | 動詞+目的語（例: 商品を注文する） |
| アクター | システムと対話する外部エンティティ |
| 前提条件 | ユースケース開始前に満たすべき条件 |
| 基本フロー | 主要な成功パス |
| 代替フロー | 基本フローからの有効な分岐 |
| 例外フロー | エラー条件による分岐 |
| 事後条件 | ユースケース完了後の状態 |

## フローの記述方法

### 基本フロー

1. アクターがアクションを実行する
2. システムがレスポンスを返す
3. （繰り返し）
4. システムが最終結果を返す

### 代替フロー

- 分岐点: 基本フローのステップ番号を参照
- 合流点: 基本フローに戻るステップ番号を明示
- 例: 「ステップ3で在庫なしの場合、代替フローA1に分岐」

### 例外フロー

- エラー条件: システムエラー、バリデーション失敗
- リカバリ: 可能な場合はリカバリパスを記述
- 例: 「ステップ5で決済失敗の場合、例外フローE1に分岐」

## アクター分析

- **主アクター**: ユースケースを開始するアクター
- **副アクター**: ユースケース実行中に呼び出される外部システム
- **権限レベル**: 各アクターの認証状態と権限

## フロー網羅基準

| 基準 | テストケース数 | 説明 |
| --- | --- | --- |
| 基本フロー | 1 | 正常系の最短パス |
| 全代替フロー | 代替フロー数 | 各代替フローを1回 |
| 全例外フロー | 例外フロー数 | 各例外フローを1回 |
| 全パス | パス数 | 全フローの組合せ（高コスト） |
