# Exploratory Testing Guide

- Purpose: 探索的テストの詳細ガイド
- Scope: SBTM、テストチャーター、探索戦略、セッションレポート

## 概要

探索的テストは、テスト設計と実行を同時に行う技法である。テスターの知識と直感を活かし、計画的テストでは発見しにくい欠陥を検出する。

## Session-Based Test Management（SBTM）

### セッションの構成

| 要素 | 説明 |
| --- | --- |
| チャーター | テストの目的と範囲 |
| タイムボックス | 制限時間（25/45/90分） |
| セッションノート | 探索中の記録 |
| バグレポート | 発見した欠陥 |
| デブリーフ | 振り返りと次のアクション |

### タイムボックスの選択

| 時間 | 用途 |
| --- | --- |
| 短（25分） | 焦点を絞った特定機能の探索 |
| 通常（45分） | 標準的な機能探索 |
| 長（90分） | 広範な統合テスト、複雑なシナリオ |

## テストチャーターの記述方法

### 形式

```text
[対象]を探索して[リスク/目的]を発見する、[手段/戦略]を使用して
```

### 例

- 「ログイン機能を探索してセキュリティ脆弱性を発見する、不正な入力を使用して」
- 「注文フローを探索して状態不整合を発見する、途中中断とリトライを使用して」
- 「API境界を探索してデータ整合性の問題を発見する、大量データ投入を使用して」

## 探索戦略（ツアーリスト）

### ガイドブックツアー

- ドキュメントに記載された機能を順に確認
- 仕様と実装の乖離を検出

### マネーツアー

- 最も重要な機能（収益に直結する機能）に集中
- ビジネスクリティカルなフローを探索

### ランドマークツアー

- 主要画面/エンドポイントを順に訪問
- 各ランドマーク間のナビゲーションを確認

### アリーツアー

- 最も使われていない機能、隠れた設定を探索
- コーナーケースの発見

### ゲリラツアー

- 短時間で攻撃的にテスト
- 壊しにいく意図で操作

### 夜のツアー

- 低負荷時・メンテナンス時の動作を探索
- バッチ処理、定期タスクの確認

## セッションレポートの形式

```text
チャーター: [テストチャーター]
開始時刻: [HH:MM]
終了時刻: [HH:MM]
テスター: [名前]

探索した領域:
- [領域1]: [概要]
- [領域2]: [概要]

発見事項:
- [BUG-001]: [概要] (severity: high/medium/low)

未探索領域:
- [領域3]: [理由]

次のアクション:
- [アクション1]
```

## 計画的テストとの使い分け

| 観点 | 計画的テスト | 探索的テスト |
| --- | --- | --- |
| 仕様の明確さ | 明確 | 不明確または未知 |
| 再現性 | 高い | 中程度（ノート依存） |
| 発見する欠陥 | 仕様ベース | 仕様外・想定外 |
| 適用タイミング | 要件確定後 | いつでも（特に初期） |
