# Exploratory Testing

- Purpose: 探索的テストによるテストケース導出
- Scope: 単一テスト技法の適用

## 実行手順

1. **テストチャーターの定義**
   - 対象機能の特定
   - テストの目的（何を探索するか）
   - リスク領域の明示
   - チャーター形式: 「[対象]を探索して[リスク]を発見する、[手段]を使用して」

2. **セッションの時間枠設定（タイムボックス）**
   - 短時間セッション（25分）: 焦点を絞った探索
   - 通常セッション（45分）: 標準的な探索
   - 長時間セッション（90分）: 広範な探索

3. **探索戦略の選択**
   - ガイドブックツアー: ドキュメントに記載された機能を順に探索
   - マネーツアー: 最も重要な機能に集中
   - ランドマークツアー: 主要画面/エンドポイントを順に確認
   - アリーツアー: 最も使われていない機能を探索
   - ゲリラツアー: 短時間で攻撃的にテスト

4. **発見事項に基づくテストケースの作成**
   - 探索中に発見したリスクや異常を体系的なテストケースに変換
   - 再現手順を明確に記録

5. **セッションノートの記録**
   - 探索したエリア、発見事項、未探索エリアを記録
   - 追加探索の推奨事項を記載

## 自己修正

- **探索範囲が広すぎる**: チャーターを再定義してスコープを絞る
- **再現性の確保が困難**: ステップをより詳細に記録
- **発見事項が少ない**: 別の探索戦略に切り替え
