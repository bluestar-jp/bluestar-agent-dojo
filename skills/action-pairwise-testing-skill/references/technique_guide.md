# Pairwise Testing Guide

- Purpose: ペアワイズテストの詳細ガイド（ISTQB準拠）
- Scope: 組合せ表の生成、制約処理、カバレッジ分析

## 概要

ペアワイズテスト（オールペア法）は、すべてのパラメータペア間の値の組合せを少なくとも1回テストする技法である。全組合せテストに比べてテストケース数を大幅に削減できる。

## 適用条件

- パラメータ数が3以上
- 全組合せテストが現実的でない（組合せ数が多すぎる）
- パラメータ間の2因子間相互作用が主要な欠陥源

## 組合せ数の比較

| パラメータ | レベル数 | 全組合せ | ペアワイズ |
| --- | --- | --- | --- |
| 3 | 各3 | 27 | 9-12 |
| 4 | 各3 | 81 | 9-12 |
| 5 | 各3 | 243 | 12-15 |
| 10 | 各3 | 59,049 | 15-20 |

## 直交配列

- 数学的に最適な組合せ表
- すべてのペアが均等に出現
- 標準的な配列サイズ（L9, L16, L25等）に合わせる必要がある

## オールペアアルゴリズム

- 貪欲法やメタヒューリスティクスで準最適な組合せ表を生成
- 直交配列より柔軟（パラメータ数やレベル数に制約なし）
- ツール活用を推奨

## 制約処理

### 無効な組合せの除外

- 技術的に不可能な組合せ（例: OS=Windows + ブラウザ=Safari）
- ビジネスルールで禁止される組合せ
- 除外後もペアワイズカバレッジを維持する必要がある

### 制約の記述

```text
IF OS = "iOS" THEN Browser != "IE"
IF Payment = "CreditCard" THEN Amount >= 100
```

## N因子カバレッジ

- 2因子（ペアワイズ）: 標準。欠陥の70-85%を検出
- 3因子: 高リスクシステム。テストケース数は増加するが検出率向上
- 選択基準: システムのリスクレベルに応じて決定
