# 昇格カリキュラム (Curriculum)

弟子（`deshi/`）が師範（`shihan/`）として認められるための審査基準である。

## 審査項目

### 1. 掟の遵守 (Constitution Check)
- [ ] `shihan/rules/Constitution.md` の内容と矛盾する指示が含まれていないか。
- [ ] 特にセキュリティ、プライバシーに関するガードレールが機能しているか。

### 2. 専門性の確立 (Role Definition)
- [ ] 役割が明確で、他の師範と責務が重複していないか。
- [ ] 「何でも屋」になっていないか。（汎用的なタスクはテックリード等に任せるべき）

### 3. 段階的開示の実装 (Progressive Disclosure)
- [ ] ユーザーに対して一度に長大な回答をしていないか。
- [ ] まず概要を伝え、詳細を尋ねるステップを踏んでいるか。

### 4. 運用実績 (Track Record)
- [ ] 実際のプロジェクトで利用され、有用性が確認されているか。
- [ ] ユーザーからのフィードバックにより、プロンプトが改善されているか。

## 昇格フロー
1. `deshi/candidates/` にプロンプトを配置し、Pull Requestを作成。
2. 既存の師範（管理者）によるレビュー。
3. マージ後、`shihan/agents/` へ移動し、`makimono/catalog.md` を更新。
