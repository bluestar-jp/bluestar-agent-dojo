# カスタムエージェントスキルのベストプラクティス (Best Practices for Custom Agent Skills)

AIエージェントのスキルを設計・構築するための、プラットフォームに依存しない本質的な原則集です。

## 1. 認知負荷の最小化 (Minimize Cognitive Load)

エージェントのコンテキストウィンドウと推論能力は有限のリソースです。

- **Less is More**: エージェントは既に一般的な知識を持っています。スキルでは「そのタスク特有の未知のコンテキスト」のみを提供してください。
- **簡潔な指示**: 形容詞や丁寧語を省き、明確な命令形を使用します。
- **段階的開示 (Progressive Loading)**: すべての情報を最初に読み込ませず、必要になった時点で詳細ドキュメント（参照ファイル）を読み込むよう指示します。

## 2. 実行の確実性 (Execution Reliability)

エージェントに「コードを書かせる」のではなく、「ツールを使わせる」ことを優先します。

- **ツールへのオフロード**: 複雑な計算やロジックは、エージェントに推論させるのではなく、検証済みのスクリプト（Utility Scripts）として提供し、それを実行させます。これにより、予測可能性と成功率が劇的に向上します。
- **計画と検証 (Plan & Verify)**: いきなり実行させず、必ず「計画の提示」→「ユーザーの合意」→「実行」→「自己検証」のループを回させます。

## 3. 自律性の制御 (Autonomy Management)

タスクの性質に応じて、エージェントの自由度を適切に制御します。

- **High Autonomy**: 創造性や柔軟な判断が必要な場合（例：アイデア出し、デバッグ）。目的と制約のみを与えます。
- **Low Autonomy**: 定型作業や正確性が最優先される場合（例：デプロイ作業）。厳格な手順書やスクリプトを与えます。

## 4. コンテキストの独立性 (Context Independence)

- **時間に依存しない**: 「今日」「現在のバージョン」といった変化する情報はスキル定義にハードコードせず、動的なツール実行や外部入力から取得させます。
- **モジュール設計**: スキルは他のスキルと依存関係を持たず、単独で機能するように設計します。

## 5. フィードバックループ (Feedback Loops)

- **自己修正**: エラーが発生した場合のリカバリ手順や、出力が要件を満たしているか確認するチェックリストを指示に含めます。

## 6. 命名規則 (Naming Convention)

- **構造**: `[type]-[action]-skill` の形式を推奨します。
- **Action**: 「何をするか」を動名詞形（verb-ing）で記述します。
- **Suffix**: 末尾に `-skill` を付与することで、ディレクトリ構造の中で「能力（Skill）」であることを明示します。
- **Example**: `proc-creating-skills-skill`, `action-reviewing-code-skill`
