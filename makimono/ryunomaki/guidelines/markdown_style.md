# Markdown Style Guide

本プロジェクトにおけるMarkdownファイルの記述ルールです。
可読性とメンテナンス性を保つため、以下のガイドラインに従ってください。

## 基本ルール

1. **ファイル名**: スネークケース（例: `my_document.md`）を使用する。
2. **エンコーディング**: UTF-8 (BOMなし)。
3. **改行コード**: LF。

## 構文スタイル

### 見出し (Headers)

- **ATX形式**（`#` を使う形式）を使用する。
- 見出しの前後には空行を入れる。
- `#` とタイトルの間には半角スペースを入れる。

```markdown
# 見出し1

## 見出し2
```

### リスト (Lists)

- **順序なしリスト**: ハイフン `-` を使用する（アスタリスク `*` は使用しない）。
- **順序付きリスト**: 数字とピリオド `1.` を使用する。
- インデントはスペース2つまたは4つで統一する。

```markdown
- 項目1
- 項目2
  - サブ項目
```

### コードブロック (Code Blocks)

- フェンスコードブロック（```` ``` ````）を使用する。
- 開始フェンスの後ろには必ず**言語識別子**を記述する。

````markdown
```python
print("Hello, World!")
```
````

### 強調 (Emphasis)

- **太字**: アスタリスク2つ `**text**` を使用する。
- *イタリック*: アスタリスク1つ `*text*` を使用する（日本語文書ではあまり使用しない）。

### リンクと画像

- インライン形式 `[リンクテキスト](URL)` を使用する。

## 日本語特有のルール

- **英数字と日本語の間**: 半角スペースを入れることを**推奨**する（可読性向上のため）。
  - OK: `Gemini CLI は便利です。`
  - 推奨されない: `Gemini CLIは便利です。`
- **句読点**: 「、」「。」を使用する（カンマ・ピリオドはコード内解説などを除き避ける）。

## ツールによるチェック

- プロジェクトルートの `.markdownlint.json` に基づき、linterが警告を出した場合は修正する。
